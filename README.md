## Project Reflections

### Grocery Tracker Application
The Grocery Tracker Application was developed to solve a practical problem for grocery stores: analyzing and managing sales data using C++. It tracks item purchase frequencies, generates frequency lists, visualizes data with histograms, and creates backups.

- **Strengths**: Designed a modular class structure that encapsulated core functionalities, ensuring adaptability and maintainability. Detailed design documentation enhanced focus on data management and ensured user needs were met.
- **Potential Improvements**: Enhance input validation and file I/O handling to improve robustness. Implementing more thorough error-checking and advanced data structures could make the application more scalable and user-friendly.

---

### Gaming Room Project
The Gaming Room Project is an OOP-based software application designed for "The Gaming Room," focusing on creating a multiplayer trivia game environment. It supports game room management, user participation, real-time trivia questions, and score tracking.

- **Strengths**: Developed a comprehensive software design document outlining system architecture, user flows, and functional requirements. Utilized UML diagrams to clarify component interactions and streamline the design process.
- **Potential Improvements**: Refine the user interface and enhance real-time communication features to improve user engagement. Future iterations could incorporate prototyping and agile development for iterative design refinement.

---

### ContactService Project
The ContactService project is a Java-based application providing an in-memory contact management system for a mobile application. The project emphasizes software testing with JUnit to verify functionality for adding, updating, and deleting contact information while enforcing strict field constraints.

- **Strengths**: Applied rigorous unit testing to ensure robustness. Implemented best practices for validation, encapsulation, and in-memory data management. JUnit facilitated component functionality verification.
- **Status**: The project is ongoing; potential areas of improvement will be reviewed upon completion.

---

### LinkedList Project
The LinkedList Project is a C++ application managing bid data for a municipal auction system using a singly linked list. It demonstrates my proficiency in data structures, memory management, and file handling.

- **Strengths**: Addressed challenges with environment setup on a Mac (Apple Silicon) by configuring tools like CMake and Homebrew. Enhanced program flexibility by dynamically prompting users to select files for loading. Gained deeper insights into linked lists and dynamic data management.
- **Potential Improvements**: Add advanced sorting algorithms and improve user interface for enhanced usability.

---

### Binary Search Tree Project
The Binary Search Tree Project is a C++ application that manages bid data for municipal property auctions using a binary search tree data structure. This project emphasizes algorithm efficiency, dynamic memory management, and file handling. The program allows users to load bids from CSV files, search for specific bids, insert new bids, and remove bids.

- **Strengths**: Successfully implemented recursive methods for tree traversal (in-order, pre-order, post-order), insertion, search, and removal. The project demonstrated my ability to optimize data management and design modular code using object-oriented principles.
- **Challenges**: Encountered errors during environment setup on Apple Silicon and difficulties debugging recursive tree operations. These were resolved by carefully examining code logic, ensuring proper memory allocation and deallocation, and configuring CMake for compatibility.
- **Potential Improvements**: Further optimize the removal logic to handle edge cases more effectively and enhance input validation for user-provided data.
